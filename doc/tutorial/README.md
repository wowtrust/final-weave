# FinalWeave 开发者入门教材

这不是六篇互不相干的术语说明，而是一条连续故事线：一位会 Go、第一次接触区块链的开发者，跟随 Alice 的同一笔交易，从“为什么需要可信账本”一直走到能够完成首个贡献和独立调试。

贯穿案例始终是：

```text
Alice 在 ledger-demo 提交：
KVPut("device/42/status", "online")
nonce = 18，有效高度 = [1200, 1300]
```

每一章都从这个案例提出新问题：哈希与签名怎样保护它、数据怎样可用、DAG 的边怎样变成隐式投票、FinalDAG-C 怎样给出稳定顺序、并行执行怎样保持串行等价、代码应该放在哪里、测试和故障证据怎样建立。

## 推荐顺序

| 顺序 | 章节 | 建议时间 | 读完后的成果 |
|---:|---|---:|---|
| 0 | [学习路线](00-learning-path.md) | 30 分钟 | 明确学习目标和验收标准 |
| 1 | [区块链基础](01-blockchain-foundations.md) | 3～4 小时 | 能解释哈希、证明、BFT、DAG 与最终性 |
| 2 | [交易完整生命周期](02-finalweave-transaction-lifecycle.md) | 2～3 小时 | 能逐步追踪 Alice 的交易及每阶段证据 |
| 3 | [开发环境与工程 Bootstrap](03-development-environment-and-codebase.md) | 2～3 小时 | 能从零搭建依赖方向正确的工程骨架 |
| 4 | [首次贡献教程](04-first-contribution-tutorial.md) | 1～2 天 | 能用测试驱动实现规范哈希或无状态校验 |
| 5 | [调试、测试与成长计划](05-debugging-testing-and-glossary.md) | 持续 | 能分层定位故障并规划前 30 天成长 |

建议严格按顺序阅读并完成每章练习。碰到不熟悉的名词时先根据上下文理解它在 Alice 这条旅程中的前因后果，再沿章节链接回看首次出现处或进入权威规范；本教程故意不提供脱离上下文的术语词典。

需要查阅规范原文时，从[完整设计文档中心](../README.md)进入；教程负责建立心智模型，规范文档才是实现字段、编码和状态迁移的权威依据。
